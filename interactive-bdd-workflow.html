核心洞察 — 規格即測試

BDD Analysis 的「詞入格組寫」將模糊需求轉換為精確的可執行規格。三巨頭規格確保行為模型、通訊契約、資料模型三角一致。Auto Feedback Loop 讓 AI 自動修復直到全綠，消除人工反覆。整套流程的目標：在規格層面做到零詮釋空間，在實作層面做到全自動閉環。

📚BDD Analysis 五步驟 + QA 六大戰術

五步驟 BDD Analysis 系統性地將需求轉換為可執行的 Gherkin 規格，搭配六大 QA 測試戰術確保全面覆蓋。

1. 詞 (Define DSL)

📚 定義 DSL 句型表 dsl.yml
Produces: 參數化句型

2. 入 (Apply QA Tactics)

📚 EP/BVA/DT/ST/CM/FP 六大戰術
Produces: 測試策略標籤

3. 格 (Derive Matrix)

📚 推導等價類矩陣 + 測試資料
Produces: 測試資料矩陣

4. 組 (Merge Examples)

📚 判定 Scenario 合併條件
Produces: Scenario vs Outline

5. 寫 (Write Feature)

📚 產出最終 Feature File
Produces: Gherkin 規格

QA 戰術 (6 Tactics)

📚 EP 等價劃分、BVA 邊界值、DT 決策表、ST 狀態轉移、CM 組合、FP 故障注入
Produces: 全面測試覆蓋
